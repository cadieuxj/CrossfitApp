**Why:** Peer review of ApexAI Athletics v1 scientific methodology.
**How to apply:** Use as baseline when reviewing future changes to the AI coaching pipeline.

## Critical Finding: Gemini Hallucinates Pose Landmarks

The overlay_data in the analysis prompt instructs Gemini to "estimate landmark positions and joint angles as accurately as possible" — meaning the pose skeleton overlay shown to users is not computed from MediaPipe or any actual computer vision system. It is generated by Gemini as text (JSON), which is a language model hallucinating numbers. Joint angles reported (LEFT_KNEE degrees, etc.) are model confabulations, not measurements. This is a fundamental validity failure for any quantitative biomechanical claim.

The knowledge base DOES correctly acknowledge: "Z-depth data from on-device pose estimation is unreliable and must not be used as a primary fault indicator." But this applies to the overlay display, not to the analysis itself. The analysis is done by Gemini watching the video, which is actually the more defensible modality — a vision-language model viewing a video is closer to a human coach watching film.

## Fault Detection Validity

Gemini 1.5 Pro is a general-purpose vision-language model, not a validated biomechanics assessment tool. No ground truth validation (against expert coach ratings, force plates, motion capture) has been conducted. Fault classifications (MINOR/MODERATE/CRITICAL) are LLM outputs with no calibration to injury incidence data. The CRITICAL classification in particular (which tells an athlete to stop and seek correction) carries potential liability without clinical validation.

## Knowledge Base Quality

The inline CROSSFIT_KNOWLEDGE_BASE is qualitatively reasonable — it accurately describes CrossFit movement standards (snatch, clean, squat faults). However:
- It lacks biomechanical references (no citations)
- Joint angle thresholds are stated as "population-level guidelines" without citing the literature they derive from
- The taxonomy (MINOR/MODERATE/CRITICAL) is not tied to any injury epidemiology

## Weight Estimation

The prompt asks Gemini to estimate barbell weight in kg. This is impossible from video without calibration objects. The model will hallucinate or estimate based on visual appearance of plates. This value should not be stored as data or displayed without a major disclaimer.

## Temperature and Consistency

Temperature = 0.2 is appropriate for structured output tasks. However, two analyses of the same video will not produce identical results — fault detection is non-deterministic. No inter-rater reliability metric is possible with this approach.

## Corrective Image Generation

Gemini 2.0 Flash generating "corrective posture images" is AI-generated illustration, not anatomically validated instruction. Images could show anatomically impossible or biomechanically incorrect positions. No human expert review of generated images is in the pipeline.
